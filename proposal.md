# 期末项目方案书：SZ-HK Hub (深港跨境专业生活助手)

## 1. 项目背景与意义 (Project Background)
随着深港“双城生活”的深入，跨境用户在金融、文旅活动以及政策解析（签注、通关）方面面临信息碎片化和实时性差的痛点。通用 LLM 常因缺乏最新的本地化政策、金融合规要求和实时交通数据而产生幻觉。本项目旨在构建一个垂直领域的专业 Agent，解决跨境高净值行为中的复杂决策问题。

## 2. 核心功能 (Key Features)
* **跨境金融导航**：提供基于最新 RAG 知识库的数字银行开户指南及合规建议。
* **活动/情报自动化处理**：自动解析社交媒体、活动官网的图片/文本，提取关键时间点并进行日程冲突检测。
* **实时决策支持**：集成实时汇率、口岸人流量、港铁班次等数据，规划最优过关与支付方案。

## 3. 技术架构 (Technical Architecture - *满足 A 级 4+ 技术要求*)

### 3.1 AI Agents (LangGraph)
采用多智能体协作架构：
* **Planner Agent**：负责将复杂需求（如“去西九龙打卡并参加比赛顺便开户”）拆解为逻辑子任务。
* **Verifier Agent (Reflection)**：对翻译或政策建议进行自我审查，确保符合最新法规。

### 3.2 高级 RAG 系统 (Advanced RAG)
* **数据源**：爬取香港特区政府、ZA Bank、MTR 官网及专业论坛的非公开/动态 PDF 与 FAQ。
* **评估指标**：第 10 周将提交基于 Hit Rate 和 MRR 的检索质量报告。

### 3.3 MCP Protocol (Model Context Protocol)
* **MCP Server**：开发自定义工具连接外部 API，包括实时汇率转换工具、Google Maps 路径规划工具以及本地文件操作工具。

### 3.4 Multimodal (多模态)
* 利用 Vision-LLM 解析用户上传的办卡预约截图、活动海报或复杂的过关指引图，实现自动填单建议。

### 3.5 Safety Guardrails (安全护栏)
* 建立金融与法律敏感词过滤层，严禁 Agent 提供任何涉及绕过外汇监管或不合规金融操作的建议。

## 4. TDD-for-AI 开发策略 (Testing Strategy)
本项目将编写 **20 个以上的 pytest 测试用例**，重点覆盖以下维度：
* **检索准确性测试**：验证针对特定银行政策提问时，RAG 是否返回了正确文档片段。
* **Agent 行为路径测试**：记录 Agent 处理跨境任务时的 Trace，验证其逻辑跳转是否符合预期。
* **Ollama 回退测试**：确保在断网或无 API Key 环境下，本地模型能维持基础问答功能。

## 5. 预期成果 (Expected Outcomes)
1.  **代码库**：包含干净、高效的 Python 代码，严格遵循 TDD 规范。
2.  **Web 应用**：可交互的深港生活助手门户。
3.  **技术报告**：详细记录 Prompt 调优过程、Agent 架构设计及 15+ 测试结果。
